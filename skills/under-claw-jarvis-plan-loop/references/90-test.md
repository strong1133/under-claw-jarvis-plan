# 90 · 자가진단 모드 (`/under-claw-jarvis-plan-loop test`)

입력이 `test`면 루프를 실제로 돌리지 않고 **이 자가진단**만 수행하고 종료한다.
**안전 원칙**: 읽기 전용. 프로젝트 파일 무변경. 외부 전송·배포 금지. 구현 세션을 실제로 띄우지 않는다.

## 점검 목표
1. **베이스 연동** — 베이스 `under-claw-jarvis-plan` 진입점(현재 호스트)이 존재·로드되는가(한 회차가 베이스를 완주할 수 있는가).
2. **루프 레퍼런스** — `00`~`40` + `90`이 로드되고 핵심 동작을 1줄로 설명 가능한가.
3. **3 역할 분리** — 오케스트레이터/구현/검수가 **분리된 세션**으로 생성 가능한가(현재 호스트의 C3 수단 또는 fallback).
4. **종료 게이트** — 9.5 게이트·MAX_ROUNDS·plateau 종료 조건이 정의·설명되는가.
5. **검수 분리 원칙** — 검수 세션이 구현 자기점수를 받지 않고 산출물·원요구만으로 채점하는 구조인가.

## 수행 절차 (호출은 `<loop test ...>`로 로깅)
1. **호스트·베이스 감지**: 현재 호스트(Claude/Codex/Gemini/그 외) 식별. 베이스 진입점과 베이스 `references/`
   존재 확인(없으면 ❌ — 루프는 베이스가 있어야 동작). 베이스 스킬의 `05-host-map` 기준 C1~C3 native/fallback 판정.
2. **루프 레퍼런스 확인**: `00-loop-control`/`10-orchestrator`/`20-implementer`/`30-reviewer`/`40-scoring`/`90-test`
   로드·요지 확인.
3. **역할 분리 실증(드라이런)**: 검수 세션을 **1개만** 띄워(실제 작업 없이) "나는 구현/오케스트레이터와 분리된
   세션이며 산출물·원요구만 본다"를 ACK시켜 분리 가능성을 실증. 서브에이전트 수단이 없으면 순차 분리 컨텍스트로 대체 확인.
4. **게이트·루브릭 확인**: 9.5 게이트, 4차원 배점(4/3/2/1=10), 보정 앵커, 도메인별 D3 해석을 로드·설명.
5. **종료 조건 확인**: 성공(≥9.5)/cap/plateau/BLOCK 4종 종료가 모두 정의됐는지.
