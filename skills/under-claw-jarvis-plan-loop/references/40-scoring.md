# 40 · 채점 루브릭 (Scoring Rubric) — 10점 만점 · 게이트 9.5

검수담당(→ `30-reviewer`)이 회차 산출물을 채점하는 고정 루브릭. 두 축을 4개 차원으로 펼치고
합산해 **종합점수(소수 1자리)** 를 낸다. 종료 게이트는 **9.5 초과**.

> 두 축(사용자 요구): ① **원요구사항 준수**, ② **해당 작업 프로젝트의 소스패턴 등 준수**.
> ②는 도메인에 맞게 일반화한다(아래 "도메인별 ② 해석").

## 차원·배점 (합 10.0)
| 차원 | 배점 | 무엇을 보나 |
|---|---|---|
| **D1 요구 충실도** | **4.0** | 원요구의 **모든** 항목을 빠짐없이·왜곡없이 충족했는가(누락·축소·확대 없음) |
| **D2 정확성·타당성** | **3.0** | 실제로 동작/성립하는가(코드면 정상 동작, 분석·기획이면 논리·수치·근거의 타당성) |
| **D3 관례·패턴 준수** | **2.0** | ② 축 — 프로젝트/도메인의 기존 패턴·컨벤션·표준을 따랐는가 |
| **D4 품질·단순성** | **1.0** | 과설계·중복 없이 단순·명료·유지보수 가능한가(Karpathy 단순성) |

## 보정 앵커 (각 차원을 0~만점으로 — 점수 드리프트 방지)
차원 점수는 배점에 아래 비율을 곱한다.
| 수준 | 비율 | 의미 |
|---|---|---|
| 탁월 | 1.00 | 결함 없음. 트집 잡을 게 없다. |
| 양호 | 0.90 | 사소한 흠 1개 이하. 실질 영향 없음. |
| 보통 | 0.75 | 눈에 띄는 흠 또는 부분 미충족 1~2개. |
| 미흡 | 0.50 | 핵심 항목 일부 미충족/오작동. |
| 불량 | 0.20 | 대부분 미충족 또는 잘못됨. |
| 실패 | 0.00 | 해당 차원 사실상 부재. |

> **9.5 게이트의 의미**: 4개 차원 가중상 거의 전 차원 "탁월~양호"여야 도달한다. 한 차원이라도
> "보통" 이하면 9.5에 못 미친다 — 즉 **원요구 누락이나 관례 이탈이 하나라도 남으면 통과 못 함**. 의도된 높은 바.

## 도메인별 ② 해석 (D3 관례·패턴 — "소스패턴"의 일반화)
| 도메인 | D3가 보는 "관례·패턴" |
|---|---|
| **개발/코드** | 기존 소스 아키텍처·네이밍·레이어링·에러처리·테스트 패턴, 린트/스타일 규칙 |
| **파일 생성** | 디렉토리·파일 네이밍 규약, 포맷·인코딩, 기존 문서 템플릿 |
| **분석** | 데이터 출처 신뢰성, 방법론 정합성, 지표 정의 일관성, 재현 가능성 |
| **기획** | 기획서 템플릿·섹션 구조, 용어·KPI 정의, 이해관계자 포맷 |
| **경제계획 수립** | 가정의 명시성, 계량 모델·단위 일관성, 시나리오·민감도 관례, 출처 표기 |

D1·D2·D4는 도메인 무관하게 동일하게 적용한다(요구 충실도·타당성·단순성은 어디서나 유효).

## 채점 절차
1. 원요구를 **항목 단위로 분해**한다(체크리스트화). D1은 이 체크리스트의 충족 비율로 본다.
2. 각 차원을 보정 앵커로 평가 → `배점 × 비율` = 차원 점수.
3. 4개 차원 합산 = 종합점수(소수 1자리, 반올림은 **계산 결과 그대로** — 게이트 통과를 위한 상향 반올림 금지).
4. 9.5 초과면 PASS, 아니면 FAIL + gap 리스트.

### 계산 예시
- D1 3.9(0.975) + D2 2.9(0.97) + D3 1.9(0.95) + D4 0.9(0.90) = **9.6 → PASS**
- D1 4.0 + D2 3.0 + D3 1.5(보통) + D4 0.9 = **9.4 → FAIL** (관례 이탈로 게이트 미달, gap: D3 패턴 정렬)

## 게이트·종료 (→ `00-loop-control`)
- **PASS**(>9.5): 루프 종료, 최종 산출물 확정.
- **FAIL**(≤9.5): gap 리스트를 다음 회차 구현 입력으로. 단 cap/plateau면 에스컬레이션.
- 사용자가 `--target`으로 게이트를 낮춰도(예: 9.0), **9.5가 기본 권장**임을 마감 보고에 1줄 명시.

## 반-게이밍 규칙 (reward hacking 방지)
- 점수는 **원요구 원문**에 고정한다. 구현이 만든 새 해석으로 기준을 바꾸지 않는다.
- 증거 없는 충족 주장 = 미충족. 표면적 그럴듯함이 아니라 **실제 검증 결과**로 채점.
- 같은 산출물을 두 번 통과시키지 않는다 — 회차마다 독립 재채점(이전 점수 인용 금지).
