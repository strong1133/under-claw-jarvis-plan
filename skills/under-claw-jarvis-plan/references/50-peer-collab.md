# 50 · Claude ↔ Codex 동등 협업 프로토콜 (2-pane 전용)

> 이 스킬의 멀티에이전트는 **두 모델이 동등한 공동 작업자**라는 전제를 지킨다.
> Claude와 Codex는 **함께 생각·계획·분석·구현**한다. 기능으로 위계를 나누지 않는다.
> (단독 Claude 환경에서는 이 프로토콜 대신 Agent/Workflow 서브에이전트로 같은 효과를 낸다.)

## 원칙 — 동등(parity)
- **기능 위계 금지**: "설계는 Claude, 구현은 Codex" 같은 고정 역할 분리는 쓰지 않는다.
  둘 다 이해·계획·분석·구현 전부에 참여한다.
- **분담은 업무 영역 기준**: 나눌 때는 기능이 아니라 **프로젝트/모듈/파일 경계**로 상호 합의해 나눈다
  (예: Claude=api, Codex=web — 둘 다 각 영역의 사고·설계·구현을 책임).
- **facilitator ≠ boss**: 스킬을 실행한 쪽(보통 Claude)이 tmux 루프를 진행(메시지 전송·취합)하지만,
  그것은 진행 역할일 뿐 **결정 권한 우위가 아니다.** 결론은 근거로 정한다.
- **이질적 관점이 가치**: 서로 다른 모델이라 사각지대가 다르다. 독립 분석 후 교차 검증으로 더 잡는다.

## 패턴 — 독립 병행 → 교차 검토 → 합의 합성
사고가 필요한 단계(이해·계획·분석·설계 결정)는 다음을 따른다:
1. **독립 병행**: 같은 문제를 Claude·Codex가 **각자 독립적으로** 분석/초안 작성(서로 안 보고).
2. **교환**: `claude-team-send <상대 pane> "메시지"` 로 결과를 주고받는다.
3. **교차 검토**: 상대 안을 `[REVIEW]` 로 적대적으로 검토 — 차이점·허점·놓친 케이스를 지적.
4. **합의 합성**: 두 안을 대등하게 합성. 불일치는 근거로 판정하고, 좁혀지지 않으면
   **양안 + 권장안**을 사용자에게 제시(어느 한쪽이 일방 결정하지 않는다).

## 적용 범위
- **복합 작업(council)**: 위 패턴을 이해(10)·계획(20)·구현(30)·검수(40)에 적용.
- **단순 질의**: 굳이 양쪽을 돌리지 않는다(비용·시간). 한쪽이 즉답.
- **비용 의식**: 독립 병행은 작업량이 2배다. 사고·분석·설계 결정 등 **다양성이 값어치 있는 곳**에 쓰고,
  기계적 작업은 영역 분담으로 병렬화한다.

## 통신 규약 (요약)
- 전송: `claude-team-send <세션:pane> "메시지"` (자동 Enter)
- 확인: `tmux capture-pane -t <세션:pane> -p -S -120`
- 태그: `[REVIEW]` 검토요청 / `[ASK]` 질문 / `[DIFF]` 변경의도 / `[FYI]` 공유 / `[ACK]`·`[BLOCK]`
- 동일 파일 동시 수정 금지: 변경 직전 `[DIFF]` → 상대 `[ACK]` 후 진행.
- 동일 쟁점 3회 왕복 금지 → 차이점만 정리해 결론/선택지로.

## 구성 스킬 로깅 (Claude·Codex 양쪽 모두 필수)
2-pane에서는 **Claude와 Codex 둘 다** 구성 스킬을 적용할 때마다 **호출 직전 한 줄 로깅**한다.
주체를 함께 표기한다: `<{태그} 호출 · CLAUDE>` / `<{태그} 호출 · CODEX>`.
| 적용 모듈 | 로깅 태그 |
|-----------|-----------|
| 00-karpathy | `<karpathy 호출>` |
| 10-understand (Understand-Anything) | `<understand-anything 호출>` |
| 20-plan (Superpowers) | `<superpowers:plan 호출>` |
| 30-implement (Superpowers) | `<superpowers:implement 호출>` |
| 40-review (Superpowers) | `<superpowers:review 호출>` |
| 50-peer-collab | `<peer-collab 적용>` |
- 예: Codex가 자기 영역을 구현하면 `<superpowers:implement 호출 · CODEX>`, 상대 안을 리뷰하면
  `<superpowers:review 호출 · CODEX>`. **호출 로깅 생략 불가**(결과 로깅은 선택).
- 프로젝트 환경 스킬(understand·code-review 등)은 구성 스킬이 아니므로 로깅 대상 아님.
